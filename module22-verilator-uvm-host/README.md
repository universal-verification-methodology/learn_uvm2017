# Module 22: Verilator as a UVM host

**Kind:** `bridge` · Activity: Verilator + Accellera UVM 2017 literacy · **Track A focus**

[← VIP anatomy](../module21-vip-anatomy/README.md) · [Course README](../README.md) · [Verilator + UVM Makefile knobs →](../module23-verilator-uvm-makefile/README.md)

## Outcomes

After this module you can explain **why Accellera UVM 2017 can ride on Verilator**, how that differs from classic event-driven SystemVerilog simulation, and what practical limits to expect before you touch a Makefile.

## Bridge (no browser lab required)

1. Open [EXAMPLES.md](EXAMPLES.md) and read `examples/verilator-host-sketch/host.txt`.
2. Restate in one paragraph: compile-to-C++ host vs event-driven SV sim, and one limit you will watch for.
3. Optional literacy refresh (browser, not graded here): [`sim-pipeline`](http://127.0.0.1:8080/tools/sim-pipeline/index.html) · [`iverilog-vs-verilator`](http://127.0.0.1:8080/tools/iverilog-vs-verilator/index.html)
4. For Verilator **tool** depth without UVM, see [`../learn_verilator/`](../learn_verilator/).
5. Continue to module 23 for Makefile knobs, then module 24 for a real run.

## Track B

None required. Browser sketches through module 21 remain concept literacy; modules **22–24** are Track A / offline fidelity.
