Constrained random verification sketch (literacy)
=================================================

Pattern on a class or seq_item:
  class MyTxn extends uvm_sequence_item;
    rand bit [7:0]  data;
    rand bit [15:0] addr;

    constraint c_range {
      data inside {[0:15]};
    }

    constraint c_align {
      addr % 4 == 0;
    }
  endclass

  MyTxn txn = MyTxn::type_id::create("txn");
  if (!txn.randomize())
    `uvm_fatal("RAND", "randomize failed")
  // txn.data, txn.addr now hold legal sample values

Steps:
  1) declare rand fields
  2) write constraint { ... } for legal region
  3) call randomize() → 1 = ok, 0 = fail (unsat / solver error)
  4) use sampled values in sequence / driver

Seed (reproducibility):
  initial begin
    void'(std::randomize(seed));  // or +SEED plusarg
  end
  // same seed + same constraints → same stream (debug flakiness)

Conflict example (randomize → 0):
  constraint c_a { x inside {[0:3]}; }
  constraint c_b { x inside {[12:15]}; }
  // empty intersection → no legal x

Common mistakes:
  conflicting hard constraints
  forgetting to check randomize() return value
  assuming randomize success = coverage done
  no seed when debugging intermittent failures

UVM tie-in:
  seq_item.randomize() before start_item
  plus SEED plusarg from module 11 for replay
