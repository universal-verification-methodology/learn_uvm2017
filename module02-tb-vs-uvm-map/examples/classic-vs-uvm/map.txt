Classic testbench ↔ UVM map (literacy — six pairs)
==================================================

1. Pin wiggle          ↔  Driver + virtual interface
   Classic: initial / task drives DUT pins with delays
   UVM:     driver pulls seq_items; drives via vif

2. Hard-coded vectors  ↔  Sequence + sequencer
   Classic: fixed lists / nested tasks per scenario
   UVM:     sequences generate items; sequencer → driver

3. $display peek       ↔  Monitor + analysis
   Classic: TB reads wires and prints in same block
   UVM:     passive monitor → analysis port / TLM

4. Inline expect       ↔  Scoreboard
   Classic: if (got !== exp) $error in stimulus block
   UVM:     predict vs actual compare on transactions

5. Flat module tb      ↔  Env + agent(s)
   Classic: one wrapper — clock, stim, check, $finish
   UVM:     test → env → agent (sqr/drv/mon) → DUT

6. Copy-paste TB       ↔  Factory / VIP reuse
   Classic: fork whole tb.v for next block / protocol
   UVM:     type overrides + packaged agents (VIP)

UART vignette (browser lab starter):
  send_byte task     ↔  uart sequence item
  $display on tx     ↔  uart monitor item
  if (rx !== byte)   ↔  scoreboard byte compare
