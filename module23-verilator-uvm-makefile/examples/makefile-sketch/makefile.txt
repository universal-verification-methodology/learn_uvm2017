Verilator + UVM Makefile knobs (literacy)
=========================================

Practice surface:
 courses/learn_uvm2017/examples/verilator-uvm-hello/
 module1/tests/uvm_tests/Makefile
 (legacy archive — fidelity-only; commands documented here)

Four knobs you will meet repeatedly:

1. UVM_HOME
 - Points at Accellera UVM library src (uvm_pkg lives here)
 - Course tree example:
 export UVM_HOME=$PWD/tools/uvm-2017/1800.2-2017-1.0/src
 - Unset → early make failure (missing uvm_pkg / includes)

2. Include paths
 - Makefile adds +incdir+ for UVM src and local TB dirs
 - Read the Makefile header — do not guess paths

3. SIM=verilator
 - Selects the Verilator compile/run recipe
 - Commercial flows use the same TEST with SIM=vcs, SIM=questa, etc.

4. TEST= / +UVM_TESTNAME
 - TEST= names the test the Makefile builds
 - Binary run often needs +UVM_TESTNAME=… so UVM picks the test class
 - Mismatch → “no tests” or UVM fatals

Typical command shape (module 24 executes this for real):

 cd courses/learn_uvm2017/examples/verilator-uvm-hello
 export UVM_HOME=$PWD/tools/uvm-2017/1800.2-2017-1.0/src
 cd module1/tests/uvm_tests
 make run SIM=verilator TEST=test_and_gate_uvm

This module (23): dry-run first

 make -n run SIM=verilator TEST=test_and_gate_uvm

Common fail modes:
 - UVM_HOME wrong or unset
 - make from wrong directory
 - verilator not on PATH
 - TEST vs +UVM_TESTNAME mismatch
 - first cold compile takes many minutes (normal)
