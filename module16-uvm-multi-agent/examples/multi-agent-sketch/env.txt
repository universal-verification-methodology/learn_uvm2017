Multi-agent UVM env sketch (literacy)
=====================================

Hierarchy:
  uvm_test
    └── uvm_env
          ├── uart_agent   (active/passive per is_active)
          │     ├── sequencer / driver / monitor
          │     └── vif → UART interface
          ├── spi_agent
          │     ├── sequencer / driver / monitor
          │     └── vif → SPI interface
          └── scoreboard (shared)
                ├── actual_imp from uart monitor.analysis_port
                └── actual_imp from spi monitor.analysis_port
                (or separate expected/actual per stream)

build_phase (env):
  uart_agent = uart_agent::type_id::create("uart_agent", this);
  spi_agent  = spi_agent::type_id::create("spi_agent",  this);
  scoreboard = my_scoreboard::type_id::create("scoreboard", this);

connect_phase (env):
  uart_agent.monitor.analysis_port.connect(scoreboard.uart_imp);
  spi_agent.monitor.analysis_port.connect(scoreboard.spi_imp);
  // driver/sequencer TLM inside each agent as usual

ConfigDB (typical — per agent path):
  uvm_config_db#(virtual uart_if)::set(this, "uart_agent*", "vif", uart_vif);
  uvm_config_db#(virtual spi_if)::set(this, "spi_agent*",  "vif", spi_vif);
  uvm_config_db#(uvm_active_passive_enum)::set(this, "uart_agent", "is_active", UVM_ACTIVE);
  uvm_config_db#(uvm_active_passive_enum)::set(this, "spi_agent",  "is_active", UVM_PASSIVE);

Why multi-agent:
  DUT has multiple interfaces — one agent each
  shared scoreboard sees traffic from all monitors
  virtual sequences (next module) coordinate parallel stimulus

Common mistakes:
  two protocols crammed into one agent
  forgot to connect one monitor's analysis_port
  one global is_active for unlike agents
  scoreboard expects one stream but two unrelated protocols mixed
