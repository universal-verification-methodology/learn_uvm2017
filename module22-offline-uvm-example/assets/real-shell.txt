Offline UVM run sketch (literacy)
=================================

Goal:
  Build and run one course UVM example outside the browser
  (Verilator + Accellera UVM 2017, or commercial SIM=…)

Practice surface:
  courses/learn_uvm2017_sv_verilator/
  (legacy Makefile / scripts tree linked from this curriculum)

Minimal flow:
  1. cd courses/learn_uvm2017_sv_verilator
  2. export UVM_HOME=$PWD/tools/uvm-2017/1800.2-2017-1.0/src
     (or your site UVM install)
  3. cd module1/tests/uvm_tests
  4. make run SIM=verilator TEST=test_and_gate_uvm

What you should see:
  Verilator compile of uvm_pkg.sv + DUT + test
  binary run with +UVM_TESTNAME=…
  UVM info / report summary — capture pass or fail

Alternate entry:
  ./scripts/module1.sh --uvm-tests
  (orchestrator sets UVM_HOME and runs selected tests)

Commercial shape (same TEST idea):
  make run SIM=vcs TEST=test_and_gate_uvm
  make run SIM=questa TEST=test_and_gate_uvm

Notes:
  First UVM+Verilator compile can take several minutes
  Browser sketches ≠ simulator pass
  Record simulator version + UVM version in your notes

Common mistakes:
  UVM_HOME unset
  make from wrong directory
  expecting instant compile on cold cache
  treating platform HTML as the offline run

# Makefile for UVM testbenches
# Supports Verilator and commercial simulators

# Default simulator
SIM ?= verilator

# Test selection
TEST ?= test_and_gate_uvm

# UVM home (set via environment or default)
UVM_HOME ?= $(shell echo $$UVM_HOME)

# Verify UVM_HOME is set
ifeq ($(UVM_HOME),)
    $(error UVM_HOME is not set. Please set it or use --uvm-home option)
endif

# Verilator flags
# Note: Using flags recommended for UVM 2017 support in Verilator
# See: https://antmicro.com/blog/2025/10/support-for-upstream-uvm-2017-in-verilator

$ make -n run SIM=verilator TEST=test_and_gate_uvm
echo "Compiling with Verilator..."
echo "Note: Using Verilator with UVM 2017 support"
echo "See: https://antmicro.com/blog/2025/10/support-for-upstream-uvm-2017-in-verilator"
echo "Note: Full UVM support requires Verilator 5.036+ with recent commits"
echo "      If compilation fails, you may need to build Verilator from latest source"
verilator -sv --timing --trace -Wno-fatal -Wno-TIMESCALEMOD -Wno-WIDTHTRUNC -I/mnt/d/proj/designs/learning/courses/learn_uvm2017_sv_verilator/tools/uvm-2017/1800.2-2017-1.0/src +incdir+/mnt/d/proj/designs/learning/courses/learn_uvm2017_sv_verilator/tools/uvm-2017/1800.2-2017-1.0/src +define+UVM_NO_DPI \
		--binary \
		/mnt/d/proj/designs/learning/courses/learn_uvm2017_sv_verilator/tools/uvm-2017/1800.2-2017-1.0/src/uvm_pkg.sv \
		../../dut/simple_gates/*.v test_and_gate_uvm.sv \
		--top-module test_and_gate_uvm
echo "Building with parallel jobs (VM_PARALLEL_BUILDS=1)..."
echo "Note: UVM generates many template instantiations, compilation may take several minutes"
make -j8 -C obj_dir -f Vtest_and_gate_uvm.mk VM_PARALLEL_BUILDS=1
echo "Compilation complete!"
echo "Running test_and_gate_uvm..."
./obj_dir/Vtest_and_gate_uvm +UVM_TESTNAME=test_and_gate_uvm
